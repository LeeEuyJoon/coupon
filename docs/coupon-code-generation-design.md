## 설계 결정

### 형식
- **길이**: 16자리
- **문자 집합**: 대문자 알파벳 + 숫자
- **혼동 문자 제외**: 
  - O (대문자 O)
  - I (대문자 I)
  - L (대문자 L)
  - 0 (숫자 0)
  - 1 (숫자 1)
- **사용 문자**: 32종

```
숫자: 2 3 4 5 6 7 8 9            (8종, 0과 1 제외)
알파벳: A B C D E F G H J K L M  (24종, O와 I 제외)
        N P Q R S T U V W X Y Z
```

### 생성 방식
위 32종 문자 중 16자리를 SecureRandom으로 무작위 추출

```java
private static final String CHARACTERS = "23456789ABCDEFGHJKLMNPQRSTUVWXYZ";
private static final int CODE_LENGTH = 16;
private static final SecureRandom RANDOM = new SecureRandom();

public static String generate() {
    StringBuilder sb = new StringBuilder(CODE_LENGTH);
    for (int i = 0; i < CODE_LENGTH; i++) {
        sb.append(CHARACTERS.charAt(RANDOM.nextInt(CHARACTERS.length())));
    }
    return sb.toString();
}
```

### 저장 형식 vs 표시 형식

| 구분 | 형식 | 예시 |
|---|---|---|
| DB 저장 | 16자리 원문 (하이픈 없음) | `2A3B4C5D6E7F8G9H` |
| 사용자 표시 | 4자리마다 하이픈 | `2A3B-4C5D-6E7F-8G9H` |

하이픈은 가독성을 위한 표시 형식  
DB에는 원문만 저장

## 충돌 확률 수학적 검증

### 전체 공간 크기

문자 종류 32종, 코드 길이 16자리:

```
N = 32^16 = (2^5)^16 = 2^80
  ≈ 1,208,925,819,614,629,174,706,176
  ≈ 1.21 × 10^24
```

약 100해의 120배 ...  
우주의 수명을 200억년으로 잡고, 1초에 100개의 쿠폰을 생성 한다고 했을 때, 우주의 생애가 약 2만번 반복 되어야 모든 쿠폰 공간을 채울 수 있는 수준

### 100만 건 생성 시

```
n = 1,000,000 = 10^6
N = 2^80 ≈ 1.21 × 10^24

n² / 2N = (10^6)² / (2 × 1.21 × 10^24)
        = 10^12 / 2.42 × 10^24
        ≈ 4.1 × 10^-13
```

**P(충돌) ≈ 4.1 × 10^-13**

이는 약 2조 4천억 분의 1. 번개에 맞을 확률(약 10^-6)보다 수백만 배 낮음

### 충돌 확률 50% 도달 기준

```
n_50% ≈ √(2N × ln2)
       ≈ √(2 × 2^80 × 0.693)
       ≈ 1.18 × 10^12
       ≈ 1조 1천억 건
```

100만 건 생성은 50% 충돌 임계점의 **약 100만 분의 1** 수준

### 요약

| 생성 수량 | 충돌 확률 |
|---|---|
| 100만 건 | ≈ 4.1 × 10^-13 (사실상 0) |
| 10억 건 | ≈ 4.1 × 10^-7 (100만 분의 1) |

**결론**: 생성 단계에서 고유성을 보장하지 않고, 순수 랜덤 생성으로도 중복 걱정 없음
